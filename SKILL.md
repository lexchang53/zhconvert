# zhconvert

## 💎 核心定位與價值

本技能專門用於將帶有簡體中文的文件批次且無痛地轉換為**符合台灣用語習慣的繁體中文**。它整合了「繁化姬 (zhconvert.org)」的線上 API，能提供極高精確度的字元與詞彙轉換，同時保留文件原本的樣式與排版。

---

## 觸發情境

當使用者提出以下需求時，應呼叫本技能：

- **簡轉繁 / 繁轉簡**：例如「幫我把這個資料夾下的簡體 Markdown 轉成繁體中文」或「把這份繁體契約轉成簡體中文」
- **台灣化 / 台灣用語轉換**：例如「將這份簡體 Word 檔案轉成台灣用語習慣的繁體」
- **香港化 / 香港用語轉換**：例如「將簡體或台灣繁體檔案轉換為符合香港習慣的繁體」
- **批次繁簡/地區詞轉換**：例如「批次轉換這幾份簡體 PPTX 簡報與 Excel 試算表為繁體」
- 支援副檔名：`.md`、`.txt`、`.docx`、`.xlsx`、`.pptx`

*注意：本技能的觸發不包含特定的英文單字，僅以中文意圖觸發。*

---

## 核心原則

1. **格式絕對保留**：
   - 純文字/Markdown：啟用 API 的標記忽略機制，保護 Markdown 語法（代碼塊、連結、粗體等）不被破壞。
   - Word/Excel/PPTX：僅提取文字送去轉換並替換，絕不修改原本的字型、大小、顏色、框線、表格樣式及動畫版面。
2. **安全第一（備份覆蓋模式）**：
   - 預設會自動將原始簡體檔案重新命名備份為 `[原檔名]_old.[副檔名]`。
   - 轉換後的繁體內容則會直接寫回原本的檔名 `[原檔名].[副檔名]`，確保原本的文件結構與連結（特別是 Markdown 內部的交叉連結）不會因為檔名變更而失效。
   - 只有在使用者明確指定 `--overwrite` 參數時，才會直接覆蓋，不產生 `_old` 備份檔。
3. **無縫環境**：
   - 腳本執行時會自動檢查 `openpyxl` (Excel) 與 `python-pptx` (PPTX) 是否安裝，若無則會自動嘗試在背景安裝，無需使用者操心。
4. **多模式支援**：
   - **預設模式 (Taiwan)**：簡轉繁 + 台灣用語在地化（例如將「合同」轉成「契約」；「信息」轉成「資訊」）。
   - **純字元轉換模式 (Traditional)**：僅將簡體字轉為繁體字，保留原本大陸用語（不作詞彙替換）。
   - **簡體化模式 (Simplified)**：純繁轉簡，不進行用語轉換（例如保留「契約」、「資訊」、「軟體」）。
   - **中國化模式 (China)**：繁轉簡 + 大陸用語在地化（例如將「契約」轉成「合同」；「資訊」轉成「信息」；「軟體」轉成「軟件」）。
   - **香港化模式 (Hongkong)**：簡/繁轉香港繁體 + 香港用語在地化。

## 📋 執行工作流

### Step 1：確認檔案範圍與目標模式
- 仔細檢查使用者提供的輸入路徑是單一檔案還是資料夾。
- 判斷目標模式：若使用者未特別說明，預設使用 `Taiwan` 模式；若要求「保留大陸用語」，使用 `Traditional` 模式。
- 判斷是否需要遞迴處理。若是資料夾，先掃描出所有支援的檔案類型（`.md`、`.txt`、`.docx`、`.xlsx`、`.pptx`），並排除檔名包含 `_old` 的備份檔。

### Step 2：🔴 CHECKPOINT
- **在執行任何轉換指令之前**，必須向使用者列出即將處理的檔案清單以及所選用的轉換模式（例如 `Taiwan` 模式），並等待使用者確認。
- **格式要求**：以條列式清單呈現，並詢問「是否開始轉換？」。得到使用者明確同意後，方可進入 Step 3。

### Step 3：執行轉換腳本
- 呼叫底層腳本執行轉換。
- **執行指令**：
  ```powershell
  python "scripts/convert.py" "<檔案或資料夾絕對路徑>" --mode "<轉換模式>"
  ```
  *(說明：預設會將原檔案重新命名備份為 `*_old.*`，若使用者明確要求不備份，可加上 `--overwrite` 參數。)*

### Step 4：驗證並回報結果
- 檢查腳本的輸出紀錄。
- 列出已成功轉換的檔案清單（包括新檔名），以及是否有任何略過或失敗的檔案。

## ⚠️ 失敗處理機制

若在執行過程中遇到以下異常，請依對應步驟進行恢復或 fallback：

| 觸發條件 (If) | 一線修復處置 (Then) | 仍失敗兜底方案 (Else) |
|---|---|---|
| **缺少必要 Python 套件** | 腳本會嘗試自動背景安裝。若卡住，請手動在 terminal 執行 `pip install python-docx openpyxl python-pptx requests`。 | 告知使用者環境缺少套件，並將失敗的錯誤訊息完整呈現給使用者協助排查。 |
| **API 連線超時或網路不通** | 腳本會回報連線錯誤。請檢查網路狀態，並建議使用者稍後重試。 | 提醒使用者繁化姬 API 暫時無法連線，可手動至網頁版轉換，或等網路恢復後再次執行。 |
| **`_old` 備份檔已存在** | 腳本會提示 `略過備份以保護原始數據`。這是正常的安全防護。 | 照常進行檔案內容轉換，並提醒使用者已存在舊備份，本次轉換結果將直接覆蓋現有的譯名檔案。 |
| **檔案權限不足或唯讀** | 腳本會報錯 `無法建立備份` 或 `Permission denied` | 提示使用者檢查該檔案是否被其他程式（如 Word、Excel）開啟，請其關閉後再重試。 |

## 🚫 禁止事項與反例

為確保文件安全與轉換正確性，請絕對不要進行以下操作：

- **不要處理不支援的副檔名**：例如舊版 Word (`.doc`)、舊版 Excel (`.xls`) 或 PDF，這些格式腳本無法處理，直接略過。
- **不要在無網路環境下強制執行**：本技能必須連線至繁化姬 API，離線狀態下無法轉換。
- **不要重複轉換已備份的檔案**：在遞迴處理目錄時，絕對不要將檔名包含 `_old` 的備份檔再次送入轉換，否則會破壞原始備份數據。
- **不要直接覆蓋原始檔（除非指定）**：除非使用者明確加上 `--overwrite`，否則一律採取「備份舊檔、產生新譯名檔」模式，以防轉檔失敗導致資料遺失。
- **不要在執行前跳過使用者確認**：批次轉換檔案前，未經 Step 2 🔴 CHECKPOINT 確認，絕不逕自執行指令。
